Angular

Framework Frontend

Angular est un framework Javascript open source basé sur typescript, développé et maintenu par Google et la communauté angular depuis 2016. Ce framework s’appuie sur le modèle MVC (Model View Controller) et permet donc de séparer les données, la logique métier et le visuel.

Son architecture se base sur des composants qu’on peut appeler et imbriquer pour créer des pages. Comme la plupart des framework actuels, Angular permet de développer des sites composés d’une seule page, ce qui permet de fluidifier la navigation en évitant le rechargement d’une page à chaque fois. On appelle ces applications des SPA : Single Page App.

Ce framework fait partie des framework les plus utilisés dans le monde, notamment grâce à son intégration d’une des plus grandes librairies css : bootstrap. C’est un choix très fréquent lorsque l’on souhaite faire des applications chargées car son architecture facilite le découpage des fonctionnalités et la structure du code.

De tous les frameworks web que j’ai utilisés, Angular est celui que je préfère, principalement en raison de son architecture. Je trouve aussi que les syntaxes sont très intuitives en ajoutant du confort au développement web sans pour autant trop s’en éloigner. Le fait d’utiliser TypeScript aide aussi à structurer le code pour bien s’y retrouver plus tard.

Anecdotes :

J’ai utilisé Angular dans une certaine quantité de projets depuis que j’ai appris ce framework en 2ème année, mais selon moi 2 projets sont importants pour cette compétence.

Le premier est Neoherba, projet fait à l’école pour apprendre ce framework, durant lequel j’ai mis en pratique beaucoup de structures fréquentes d’Angular, en développant des composants CRUD notamment et en gérant la communication du frontend avec une API Java pour la logique métier. J’ai aussi utilisé des notions plus avancées de templating pour la gestion dynamique des pages qu’il était possible d’ajouter dans le site.

Le second projet illustrant ma maîtrise de cette technologie est le projet de Refonte des outils internes durant mon alternance. Ce projet m’a demandé dans un premier temps d’effectuer un arbitrage pour choisir la technologie, ce qui m’a demandé des connaissances plus transverses et architecturales de Angular. Ensuite, lors de la réalisation des nouvelles interfaces, j’ai approfondi mon usage du framework en intégrant l’usage de directives dans mes composants pour les rendre plus indépendants et réutilisables.

Autocritique :

Angular est une de mes compétences techniques phares, c’est un framework frontend que j’apprécie et dont l’apprentissage m’a paru parfaitement naturel. Je trouve que sa syntaxe se mélange très bien aux syntaxes html tout en segmentant la logique et la structure des composants. C’est un découpage que j’apprécie plus à titre personnel que le découpage des frameworks comme React ou Vue qui regroupent souvent les deux.

Les projets que j’ai effectués durant mon parcours m’ont permis d’atteindre un niveau confirmé sur cette compétence, dont j’estime avoir acquis les connaissances théoriques nécessaires en plus de sa pratique. Comme toujours, il me reste une marge de progression assez difficile à déterminer dans ma capacité à développer des composants de plus en plus complexes, qui m’empêche de qualifier mon niveau de maîtrise d’”expert”. Cette capacité s’améliorera au fur et à mesure des défis de développement que je rencontrerai, je ne pense pas que ce soit quelque chose qui puisse s’apprendre uniquement dans une formation.
