Java / SpringBoot

Développement Backend

Java est un langage de programmation de haut niveau orienté objet présenté officiellement en 1995. Le langage a été créé par la société Sun microsystems et a été racheté par Oracle en 2009.

La particularité de Java est que son code est d’abord compilé en bytecode avant d’être exécuté par une Java Virtual Machine (JVM), ce qui permet d’avoir les mêmes comportements avec le code exécuté quel que soit l’OS sur lequel on le lance (Linux, Windows ou MacOS)

De nos jours, Java est la technologie la plus utilisée pour créer des applications orientées vers des fonctionnalités backend, telles que des API par exemple. Bien que le langage permette aussi de développer des interfaces graphiques, on préfère généralement des solutions orientées web pour ce faire en raison de leurs capacités multiplateformes.

Si Java est encore aussi populaire de nos jours, c’est en partie aussi grâce à un framework particulièrement connu dans le monde des développeurs Java : Spring

Inauguré en 2003, Spring est un framework open source permettant de construire et définir l’infrastructure d’une application Java, dont il facilite le développement et les tests.

Spring permet de rajouter une couche d’abstraction s’occupant à notre place de la gestion d’un contexte et s’occupe de la recherche de dépendances ainsi que de leur injection. Il fonctionne principalement à l’aide d’annotations et de dépendances dans le constructeur. Cette couche, en plus d’être très pratique pour les développeurs car gérer un contexte à la main peut très souvent être source d’erreurs, ne concurrence pas les autres dépendances que l’on pourrait vouloir dans un projet Java.

Avec le temps, Spring a évolué et propose maintenant un très grand nombre de sous librairies permettant la gestion de beaucoup de fonctionnalités courantes dans les projets java, pour n’en citer que quelques-unes on retrouve : Spring web pour créer des services REST, Spring Data JPA pour l’accès aux données ou encore Spring Security pour gérer les processus d’authentification.

Toutes ces fonctionnalités ont fait de Spring un de facto lorsque l’on se penche sur la création d’un projet Java, et pour avoir travaillé avec et sans Spring, je peux dire que l’essayer c’est l’adopter !

Anecdotes

J’ai utilisé Java pour beaucoup de projet durant mon parcours, mais pour moi les plus importants pour illustrer mon évolution avec la compétence ont été Neoherba et Activ-Infinite

Neoherba dans un premier temps est un projet sur lequel j’ai développé une API en Java J2EE, ce qui implique que j’ai dû écrire à la main le code servlet de l’API. Ce travail a été difficile mais il m’a permis de bien comprendre le fonctionnement d’une API et le comportement de Java dans ce cas précis.

Pour ce qui est de Activ-Infinite, ce projet a été pour moi une occasion d’une part de mettre en pratique mes connaissances existantes en Java ainsi que des concepts qui y sont associés (POO, Gestion de la données propre) et aussi de rencontrer de nouveaux défis architecturaux sur un projet à plus grande échelle. Ma plus grande réussite ici a été la refactorisation du code pour l’implémentation d’une norme de sécurité dans le projet. Cette réussite est traitée plus en détails dans la page dédiée de la réalisation.

Autocritique :

Selon moi, ma maîtrise de Java est une de mes compétences les plus importantes. J’ai eu l’occasion de mettre en pratique et de développer cette compétence tout au long de mon alternance et j’ai pu en constater l’évolution, que ce soit en matière d’autonomie ou de confiance en moi. J’ai vu graduellement augmenter ma productivité ainsi que la complexité des tickets qui m’ont été confiés, ce qui montre que mon équipe a aussi constaté et validé ma progression.

J’estime après ces années de travail avec le langage avoir acquis un niveau confirmé dans la compétence. Bien que “confirmé” semble un peu réducteur pour qualifier mon niveau de maîtrise, je ne pense pas que le niveau expert soit pertinent, c’est un niveau qui s’acquiert non pas avec des connaissances techniques ou architecturales, mais avec beaucoup d’expérience.

A l’avenir, je souhaite devenir un expert de cette technologie toujours aussi utilisée et je souhaite particulièrement progresser sur mes capacités en gestion de dépendances, plus faible aujourd'hui que le reste dans cette compétence.
